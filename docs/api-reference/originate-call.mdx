## Descripción

Este endpoint inicia una llamada saliente hacia el número de teléfono especificado. La llamada se encola de forma asíncrona — la API responde de inmediato con un `call_id` que usarás para controlar la llamada en los pasos siguientes.

<Note>
  Este es siempre el primer paso. Necesitas el `call_id` que devuelve este endpoint para poder usar `/say`, `/audio-fork-start`, `/break` o `/hangup`.
</Note>

## Flujo de la llamada (flow)

El parámetro `flow` define qué va a pasar cuando la llamada sea contestada. Se ejecuta automáticamente — no necesitas hacer ninguna petición adicional.

Cuando originas una llamada, tienes dos formas de interactuar con ella:

<CardGroup cols={2}>
  <Card title="Solo texto a voz (TTS)" icon="message">
    La API reproduce un mensaje pregrabado al destinatario. Solo necesitas enviar `text` y `voice`. Es la opción más simple.
  </Card>
  <Card title="Audio fork + agente" icon="waveform">
    El audio de la llamada se transmite en tiempo real a tu servidor via WebSocket. Ideal si tienes un agente de IA o un sistema que necesita escuchar y responder dinámicamente.
  </Card>
</CardGroup>

### Opción 1 — Mensaje de voz (TTS)

Usa `action: "say"` con los siguientes parámetros:

| Parámetro | Requerido | Descripción |
|-----------|-----------|-------------|
| `action` | Sí | Valor fijo: `"say"` |
| `text` | Sí | El texto que se reproducirá al destinatario |
| `voice` | Sí | La voz a usar (ej: `"ES-MX-FEMALE-01"`) |

```json
{
  "phone": "+14155552671",
  "flow": {
    "steps": [
      {
        "action": "say",
        "text": "Hola, le llamamos para informarle sobre su cuenta.",
        "voice": "ES-MX-FEMALE-01"
      }
    ]
  }
}
```

### Opción 2 — Agente en tiempo real (audio fork)

Usa `action: "audio_fork_start"` para transmitir el audio de la llamada a un servidor WebSocket. Ese servidor puede ser un agente de IA u otro sistema que escucha y responde de forma dinámica.

| Parámetro | Requerido | Descripción |
|-----------|-----------|-------------|
| `action` | Sí | Valor fijo: `"audio_fork_start"` |
| `wss_url` | Sí | URL del servidor WebSocket que recibirá el audio |
| `metadata` | No | Datos del lead que se envían al servidor WebSocket para que el agente los use durante la llamada (nombre, producto, contexto, etc.) |

```json
{
  "phone": "+14155552671",
  "flow": {
    "steps": [
      {
        "action": "audio_fork_start",
        "wss_url": "wss://your-agent-server.example.com",
        "metadata": {
          "nombre": "Juan Pérez",
          "producto": "seguro de vida",
          "id_lead": "lead-789"
        }
      }
    ]
  }
}
```

## Detección de contestador (VMD)

El parámetro `vmd_enabled` permite detectar si la llamada fue contestada por una persona real o por un contestador automático.

Tienes dos opciones:

- **`true`** — usa el servicio de VMD predeterminado de NJ Dialer. Es la opción más sencilla si no tienes un servicio propio.
- **`"wss://tu-servicio.com/vmd"`** — apunta a tu propio servicio de detección via WebSocket. Útil si tienes un modelo de VMD personalizado.

Si no envías este parámetro, la detección de contestador queda desactivada.

## Webhooks de estado

Si envías `webhook_url_status`, recibirás actualizaciones del estado de la llamada en esa URL a medida que avanza (marcando, contestada, colgada, etc.).

```json
{
  "phone": "+14155552671",
  "webhook_url_status": "https://mi-app.com/webhooks/llamadas"
}
```

## Ejemplos

<CodeGroup>

```bash Llamada simple
curl --request POST \
  --url https://nj-dialer-receiver.rd.l7labs.art/api/originate-call \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "phone": "+14155552671",
    "webhook_url_status": "https://example.com/webhook-status",
    "vmd_enabled": true,
```

## Respuesta exitosa

```json
{
  "request_id": "01JRX5Y2K3ABCDEF",
  "call_id": "01JRX5Y2K3GHIJKL",
  "status": "queued",
  "topic": "calls"
}
```

## Sin ruta disponible

Si no hay carrier disponible para el número destino, la API devuelve `422` con `status: "no_route"`:

```json
{
  "request_id": "01JRX5Y2K3ABCDEF",
  "call_id": "01JRX5Y2K3GHIJKL",
  "status": "no_route",
  "reason_code": "no_carrier"
}
```
