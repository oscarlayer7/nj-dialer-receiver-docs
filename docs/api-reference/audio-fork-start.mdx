title: "Iniciar audio fork"
openapi: "POST /api/audio-fork-start"
description: "Inicia la transmisión del audio de una llamada activa hacia un servidor WebSocket."
---

## ¿Qué es el audio fork?

El audio fork conecta la llamada activa a una URL WebSocket (`wss://`) de tu elección. A partir de ese momento, el audio de la llamada se transmite en tiempo real a ese servidor, que puede procesarlo como necesite: un agente de IA que responde, transcripción en vivo, grabación, etc.

```
Llamada activa  →  NJ Dialer  →  wss://tu-servidor.com  →  Agente / IA / Grabación
```

<Note>
  El servidor WebSocket debe estar escuchando antes de llamar a este endpoint. La conexión se establece en el momento en que se procesa la acción.
</Note>

## Ejemplo básico

```bash
curl -X POST https://nj-dialer-receiver.rd.l7labs.art/api/audio-fork-start \
  -H "Authorization: Bearer <tu_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "call_id": "01JRX5Y2K3GHIJKL",
    "wss_url": "wss://your-agent-server.example.com"
  }'
```

## Parámetros opcionales

**`mix_type`** — Define qué canales de audio se transmiten:

| Valor | Descripción |
|-------|-------------|
| `mono` | Audio mezclado de ambos lados (por defecto) |
| `stereo` | Canal separado por cada lado de la llamada |

**`sampling_rate`** — Frecuencia de muestreo en Hz. Valores comunes: `8000`, `16000`. Por defecto: `8000`.

**`metadata`** — Objeto JSON libre que se envía junto con el stream. Útil para que tu servidor identifique la llamada (ID de agente, campaña, etc.).

## Ejemplo con opciones

```bash
curl -X POST https://nj-dialer-receiver.rd.l7labs.art/api/audio-fork-start \
  -H "Authorization: Bearer <tu_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "call_id": "01JRX5Y2K3GHIJKL",
    "wss_url": "wss://your-agent-server.example.com",
    "mix_type": "mono",
    "sampling_rate": 16000,
    "metadata": {
      "agent_id": "agent-456",
      "campaign": "agosto-2025"
    }
  }'
```

## Respuesta

```json
{
  "request_id": "01JRX5Y2K3ABCDEF",
  "status": "queued"
}
```

<Tip>
  Para detener el stream antes de que la llamada termine, usa `/break`.
</Tip>
