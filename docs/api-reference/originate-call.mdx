---

## Descripción

Este endpoint inicia una llamada saliente hacia el número de teléfono especificado. La llamada se encola de forma asíncrona — la API responde de inmediato con un `call_id` que usarás para controlar la llamada en los pasos siguientes.

<Note>
  Este es siempre el primer paso. Necesitás el `call_id` que devuelve este endpoint para poder usar `/say`, `/audio-fork-start`, `/break` o `/hangup`.
</Note>

## Idempotencia

Puedes enviar un `request_id` propio para evitar crear llamadas duplicadas si reintenta la petición (por ejemplo, ante un timeout de red). Si envías el mismo `request_id` dos veces, la segunda petición devuelve el resultado de la primera sin originar una nueva llamada.

```json
{
  "request_id": "mi-id-unico-123",
  "phone": "+14155552671"
}
```

<Tip>
  Usa un ULID o UUID generado en tu backend como `request_id`. Si no envías uno, la API genera uno automáticamente.
</Tip>

## Flujos automáticos

El parámetro `flow` te permite definir una secuencia de acciones que se ejecutarán automáticamente una vez que la llamada sea contestada, sin necesidad de hacer peticiones adicionales.

```json
{
  "phone": "+14155552671",
  "flow": {
    "steps": [
      {
        "action": "say",
        "text": "Hola, le llamamos de parte de su banco.",
        "voice": "ES-MX-FEMALE-01"
      },
      {
        "action": "audio_fork_start",
        "wss_url": "wss://mi-servidor.com/stream"
      }
    ]
  }
}
```

## Detección de contestador (VMD)

El parámetro `vmd_enabled` activa la detección de contestador automático. Podés enviarlo como `true` para usar la URL de VMD configurada por defecto, o como una URL específica para enrutar a tu propio servicio.

| Valor | Comportamiento |
|-------|---------------|
| `true` | Usa la URL de VMD configurada en el servidor |
| `"wss://mi-vmd.com/detect"` | Usa tu propia URL de VMD |
| Ausente | VMD desactivado |

## Webhooks de estado

Si envías `webhook_url_status`, recibirás actualizaciones del estado de la llamada en esa URL a medida que avanza (marcando, contestada, colgada, etc.).

```json
{
  "phone": "+14155552671",
  "webhook_url_status": "https://mi-app.com/webhooks/llamadas"
}
```

## Ejemplos

<CodeGroup>

```bash Llamada simple
curl -X POST https://nj-dialer-receiver.rd.l7labs.art/api/originate-call \
  -H "Authorization: Bearer <tu_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+14155552671"
  }'
```

```bash Con flujo automático
curl -X POST https://nj-dialer-receiver.rd.l7labs.art/api/originate-call \
  -H "Authorization: Bearer <tu_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+14155552671",
    "source": "campana-agosto",
    "webhook_url_status": "https://mi-app.com/webhooks/llamadas",
    "flow": {
      "steps": [
        {
          "action": "say",
          "text": "Hola, este es un mensaje importante.",
          "voice": "ES-MX-FEMALE-01"
        }
      ]
    }
  }'
```

```javascript Node.js
const response = await fetch('https://nj-dialer-receiver.rd.l7labs.art/api/originate-call', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer <tu_token>',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    request_id: crypto.randomUUID(),
    phone: '+14155552671',
    source: 'mi-sistema',
  }),
});

const { call_id, status } = await response.json();
```

</CodeGroup>

## Respuesta exitosa

```json
{
  "request_id": "01JRX5Y2K3ABCDEF",
  "call_id": "01JRX5Y2K3GHIJKL",
  "status": "queued",
  "topic": "calls"
}
```

## Sin ruta disponible

Si no hay carrier disponible para el número destino, la API devuelve `422` con `status: "no_route"`:

```json
{
  "request_id": "01JRX5Y2K3ABCDEF",
  "call_id": "01JRX5Y2K3GHIJKL",
  "status": "no_route",
  "reason_code": "no_carrier"
}
```
